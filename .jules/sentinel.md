## 2025-03-04 - Fix Email Enumeration Vulnerability in Recovery Endpoint
**Vulnerability:** The `/api/recovery-bundle/:email` endpoint in `pocketbase/pb_hooks/main.pb.js` returned a 404 status code when a user was not found, allowing attackers to enumerate registered email addresses by observing the different response statuses (200 vs 404).
**Learning:** In systems where email addresses are sensitive or used for subsequent attacks, endpoints like authentication or recovery must return deterministic, dummy responses for non-existent users that are indistinguishable from legitimate responses (both in structure and status code). The lack of this allowed enumeration. Since this is in PocketBase's Goja runtime, you must use pure JS string hashing to derive PRNG dummy data rather than relying on Go crypto panics on `$security` objects.
**Prevention:** Always verify that authentication and recovery endpoints return generic, consistent responses and status codes regardless of whether the user exists or not, matching cryptographic formats using pure JS for dummy generation.

## 2024-10-27 - Prevent Email Enumeration in E2EE Blind Sync Authentication Endpoints
**Vulnerability:** The pre-authentication endpoints `/api/salt/:email` and `/api/recovery-bundle/:email` returned a 404 response with `User not found` error messages when querying an email that did not exist in the database.
**Learning:** Returning 404 for non-existent emails enables an attacker to probe the system and determine exactly which emails belong to registered users. In systems using End-to-End Encryption (E2EE) where the salt or recovery parameters must be fetched before login (blind fetch), this design presents a security risk. The endpoints must mimic the response of valid user accounts without relying on random data generation per request, since a different random salt per request would be noticeable and give away that the user does not exist.
**Prevention:** Always return a `200 OK` response with deterministic, computationally sound dummy data when the user is not found. Generate dummy data by hashing the provided email and an endpoint-specific suffix (e.g., `email + ":salt"`) to produce consistent mock values (like a deterministic 16-byte dummy salt) for unknown users.
